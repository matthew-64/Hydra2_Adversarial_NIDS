    Adversarial Attack ID Description
    Evasion: Rate An evasion attack which performs SYN floods with a low and slow packet rate of approximately 20pps.
    Evasion: Payload An evasion attack which performs SYN floods with perturbed payload sizes, with a payload of 90 bytes.
    Evasion: Pairflow An evasion attack with performs SYN floods with forged bidrectional traffic to give the appearance of legitimate communications
    Evasion: Rate+Payload An evasion attack which performs SYN floods with a low and slow packet rate of approximately 20pps and a perturbed payload size of 90 bytes
    Evasion: Rate+Pairflow An evasion attack which performs stealth SYN floods with perturbed packet rates of 20pps and forged bidirectional traffic to give the appearance of legitimate communicatio
    Evasion: Payload+Pairflow An evasion attack which performs stealth SYN floods with perturbed packet payload sizes of 90 bytes and forged bidirectional traffic to give the appearance of legitimate communications.
    Evasion: Stealth An evasion attack which performs stealth SYN floods with perturbed slow packet rates of 20pps, increased payload sizes of 90 bytes and forged bidirectional traffic to give the appearance of legitimate communications.

  • Use-case tutorial

    1. 1. Configuration – Within the configuration element of the GUI, a user configures the setup of the network within which the adversarial tests will take place. Choose your network type and topology before progressing to step 2.
    2. 2. Test Creation – Once the network is configured, the user can craft an adversarial attack using the provided selection boxes.
      a. Machine learning classifier – The choice of machine learning classifier to perform an attack against. Different classifiers have different weaknesses therefore the attacks will be individually tailored to exploit them as they classify traffic within a NIDS. Currently the tool supports Random Forest, SVM, KNN and Neural Network.
      b. Network attack – Different network attacks display differing characteristics therefore the user can choose which attack to perform that will be the basis of the adversarial attack. The system has options for a SYN Flood and a UDP Flood (DDoS).
      c. Adversarial attack – This is the adversarial method applied to the network attack. The user has a choice of numerous evasion and overstimulation methods to attempt to circumvent the NIDS classifier.
    3. 3. Test Submission – A test will only submit when all settings have been selected, otherwise the adversarial attack hasn’t been configured in full. Once a test is submitted, it will appear in element 3 of the Figure, where the result will be “pending”, and the test will be running.
    4. 4. Test Table – This table contains a list of all tests past and present within the database. If a test is currently running and hasn't returned a result, the result will read 'pending'. Hovering over the adversarial attack cells of each test will display a tooltip to provide further information on the attack.
